# WaveDance 项目上下文（v0.1）

## 1. 项目目标

构建一个可在 macOS 上运行的桌面应用，实时获取“系统正在播放的音频”（非麦克风输入），并对声音片段进行图形化示波展示。  
项目要求高解耦、高渲染性能，并具备良好的安装与分发便携性。

---

## 2. 需求拆解与约束

### 2.1 核心功能

1. **平台**：macOS（优先支持 Apple Silicon，兼容 Intel）。
2. **音频来源**：系统播放音频（Loopback/Capture），不走麦克风。
3. **可视化**：对音频流分段并生成示波图（Waveform）。
4. **架构**：模块解耦，便于后续扩展（频谱、录制、导出等）。
5. **性能**：低延迟采集 + 高帧率渲染 + 稳定内存占用。
6. **分发**：用户安装便捷（dmg/pkg），尽量降低使用门槛。

### 2.2 关键技术约束（macOS）

- macOS 没有“直接无权限读取所有系统音频”的通用 API，通常需要：
  - 使用虚拟音频设备（如 BlackHole）作为系统输出/聚合设备；
  - 或使用系统级音频回环方案（商业驱动如 Loopback）。
- 对于开源可落地路线，建议默认支持 **BlackHole**（免费、社区成熟）。

---

## 3. 总体技术路线

## 3.1 推荐技术栈（首版）

- **桌面壳**：Tauri（Rust 后端 + Web 前端）
  - 优点：包体小、性能好、跨平台潜力强（虽当前目标是 macOS）。
- **音频采集**：Rust + CoreAudio（或封装库）
  - 负责从指定输出设备（虚拟回环设备）拉取 PCM 数据。
- **可视化前端**：TypeScript + WebGL（优先）/Canvas2D（兜底）
  - 高频刷新下 WebGL 更稳，后续可扩展频谱与粒子特效。
- **进程通信**：Tauri Event / IPC
  - 音频线程与渲染线程之间使用 ring buffer + 批量事件推送。

> 备选：Electron + Node 原生模块也可做，但包体与资源占用通常高于 Tauri。

## 3.2 数据流（首版）

1. 采集层从 CoreAudio 获取 PCM（例如 float32，48kHz，双声道）。
2. 预处理层做：
   - 下采样（降低可视化计算量）；
   - 分帧切片（例如每 10~20ms 一帧）；
   - 振幅归一化与峰值提取。
3. 缓冲层 ring buffer 保存最近 N 秒可视化数据。
4. 渲染层基于 `requestAnimationFrame` 拉取最新帧并绘制波形。

## 4. 解耦架构设计

建议以“端口与适配器”思路组织模块：

### 4.1 模块划分

- `audio-capture`（采集域）
  - 职责：设备枚举、设备选择、音频流读取。
- `audio-processing`（处理域）
  - 职责：分帧、降采样、峰值/RMS 计算、段落切分。
- `visualization`（展示域）
  - 职责：示波绘制、时间轴、主题样式。
- `application`（编排域）
  - 职责：状态管理、配置持久化、命令分发。
- `platform`（平台域）
  - 职责：macOS 权限提示、安装检测、系统集成。

### 4.2 接口抽象（关键）

- `IAudioSource`：统一音频输入接口（后续可扩展为文件输入、网络流）。
- `IWaveformExtractor`：统一波形数据提取接口。
- `IRenderAdapter`：统一渲染接口（WebGL/Canvas 可替换）。
- `ISettingsStore`：统一配置存储接口（json/sqlite 可替换）。

通过接口层保证：
- 采集实现可替换；
- 算法可替换；
- 渲染技术可替换；
- UI 不直接依赖底层音频细节。

## 5. 性能设计原则

### 5.1 采集侧

- 音频回调线程只做“最小必要工作”，避免复杂计算。
- 使用 lock-free ring buffer，减少锁竞争与抖动。
- 采用批处理发送而非逐样本发送，降低 IPC 压力。

### 5.2 处理侧

- 分层缓存：原始 PCM 缓冲 + 可视化点集缓冲。
- 可视化点数按像素宽度上限约束（避免过绘制）。
- 对历史片段进行分辨率分级（LOD），缩放时按需取样。

### 5.3 渲染侧

- 优先 WebGL 线条批绘制（VBO 更新策略）。
- 将计算前置到 Rust 或 Worker，前端只做渲染拼装。
- 控制目标帧率（如 60fps；低性能机器可降到 30fps）。

### 5.4 指标目标（首版建议）

- 端到端显示延迟：< 120ms
- 常规设备 CPU 占用：< 12%
- 内存常驻：< 250MB
- 波形刷新流畅度：>= 55fps（大部分场景）

## 6. 安装与发布便携性

### 6.1 打包分发

- 提供 `.dmg`（主要）+ 可选 `.pkg`。
- 默认签名，后续支持 notarization（提升 Gatekeeper 体验）。

### 6.2 首次使用引导

- 检测 BlackHole 是否安装：
  - 未安装：给出一键跳转安装引导。
  - 已安装：引导用户将系统输出切换到对应设备。
- 在应用内提供“音频设备状态检查页”。

### 6.3 用户侧最小操作路径

1. 安装 App；
2. 安装/确认 BlackHole；
3. 选择输出设备；
4. 打开 App 即可看到实时波形。

## 7. 里程碑计划（MVP -> 可发布）

### M1：最小可用链路（1~2 周）

- 打通：系统音频采集 -> IPC -> 前端实时波形。
- 支持设备枚举与选择。
- 提供基础设置页。

### M2：性能与稳定性（1~2 周）

- ring buffer 与批处理优化。
- WebGL 渲染接入与性能调优。
- 基础异常恢复（设备断连、采样率变化）。

### M3：发布能力（1 周）

- dmg 打包、签名、安装引导文案。
- 黑盒回归测试（不同 macOS 版本 / 不同芯片）。

## 8. 验收标准（首版）

- 能稳定捕获系统播放音频（非麦克风）。
- 连续运行 30 分钟无明显内存泄漏或卡顿。
- 波形与声音变化具有明显同步性。
- 新用户按引导可在 5 分钟内完成首次使用。
